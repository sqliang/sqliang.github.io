# 基于 AI 编程 3 天实现监控数据汇总插件（油猴脚本版）

> 本文档为油猴脚本插件的初始化说明，旨在 3 天内借助 AI 编程能力完成一个可落地的「监控数据汇总」用户脚本。

***

## 1. 脚本定位

* **类型**：Tampermonkey / Greasemonkey 用户脚本
* **场景**：在指定监控平台（如 Prometheus + Grafana、自建运维面板等）页面自动抓取关键指标，汇总后推送至飞书 / 钉钉 / 企业微信机器人
* **目标**：零后端、零部署，纯前端脚本即可运行

***

## 2. 核心功能（MVP）

| 功能点 | 描述 | 优先级 |
| --- | --- | --- |
| 指标抓取 | 通过 DOM 选择器或接口轮询获取页面上的 CPU、内存、磁盘、QPS 等关键指标 | P0 |
| 数据缓存 | 使用 GM\_setValue / localStorage 做 5 min 滑动窗口缓存，避免重复计算 | P0 |
| 阈值告警 | 可配置阈值，超标时在页面顶部弹出 Banner 提醒 | P0 |
| 群机器人推送 | 超标持续 3 个周期后，将汇总数据 POST 到飞书机器人 | P0 |
| 配置面板 | 点击悬浮按钮弹出配置框，可改阈值、机器人 hook、开关 | P1 |
| 深色主题适配 | 自动识别页面主题，切换配色 | P2 |

***

## 3. 技术选型（AI 辅助）

| 模块 | 方案 | AI 提示示例 |
| --- | --- | --- |
| 网络请求 | `GM_xmlhttpRequest` 绕过 CORS | “用 tampermonkey 的 GM\_xmlhttpRequest 发 JSON POST 示例” |
| 数据解析 | XPath + 正则双保险 | “写一段油猴脚本，用 XPath 抓取 table 中第二列数字” |
| 配置持久化 | `GM_getValue / GM_setValue` | “油猴脚本如何持久化保存用户配置” |
| 前端组件 | 原生 HTML + CSS 实现悬浮窗 | “不用框架，写一个可拖拽的悬浮配置按钮” |
| 构建发布 | 直接 `.user.js` 文件，GreasyFork 一键安装 | “GreasyFork 发布脚本注意事项” |
